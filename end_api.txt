Phoenix Payment Service - Endpoints

Health (no auth)
- GET /health
- GET /ready

Payments (JWT auth required)
- POST /payments
  - Body: { bookingId, amount, currency, paymentMethod, metadata? }
- GET /payments
  - USER: returns own payments
  - ADMIN: returns own payments unless ?all=true is provided
  - Query: ?all=true (ADMIN only)
- GET /payments/:id
  - Owner or ADMIN
- PATCH /payments/:id/status
  - ADMIN only
  - Body: { status: PENDING|PROCESSING|SUCCESS|FAILED|CANCELLED|REFUNDED }
- PATCH /payments/:id/cancel
  - Owner or ADMIN

Refunds (JWT auth required)
- POST /refunds
  - Body: { paymentId, refundAmount, refundReason }
- GET /refunds/:id
  - Owner or ADMIN
- GET /refunds/payment/:paymentId
  - Owner or ADMIN
- PATCH /refunds/:id/status
  - ADMIN only
  - Body: { status: REQUESTED|APPROVED|REJECTED|COMPLETED }
